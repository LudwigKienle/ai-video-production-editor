# Visibility Playbook

This playbook is for growing visibility around AI Video Production Editor after
the source-code release.

The main goal is not to shout louder. The goal is to make the project's real
depth obvious to people who only give the repository 10 to 30 seconds.

## Primary Message

AI Video Production Editor is an open-source, local-first AI film production
workspace:

```text
script -> Director -> storyboard -> AI filming -> continuity review -> edit -> export
```

Avoid positioning it as only an "AI video editor". That sounds smaller than the
project. The stronger category is:

- open-source AI film production workspace
- local-first AI video production studio
- script-to-filming AI production OS
- Blender-inspired open workflow layer for AI video

## GitHub Checklist

- Social Preview image uploaded manually from
  `docs/assets/marketing/github-social-preview.png`.
- Repository description uses the full category:
  `Local-first open-source AI film production workspace: script, Director, storyboard, AI filming, continuity review, edit, export.`
- Topics include at least:
  `ai-video`, `video-editor`, `electron`, `react`, `filmmaking`,
  `generative-ai`, `storyboard`, `text-to-video`, `image-to-video`, `byok`,
  `local-first`, `open-source`, `ai-filmmaking`, `ai-tools`.
- README top section contains trailer, quick-start commands, capability map,
  and demo project.
- The repo has public starter issues with `good first issue`, `docs`,
  `provider-api`, `model-adapter`, and `ui-polish` labels.
- Releases should use the prepared notes in
  `docs/releases/v0.1.0-open-source-launch.md`.

## Launch Sequence

1. Upload GitHub Social Preview.
2. Pin the repository on the maintainer GitHub profile.
3. Post the LinkedIn launch story with the trailer and repo link.
4. Post a shorter product-focused version on X/Threads.
5. Submit a "Show HN" post after README and demo project are live.
6. Share in relevant AI filmmaking and open-source communities.
7. Ask for focused contributions, not generic support.

## Post Formats

### Short GitHub Pitch

```text
Open-source AI film production workspace:
script -> Director -> storyboard -> AI filming -> continuity review -> edit -> export.
Local-first, BYOK, Electron + React.
```

### Contributor Ask

```text
Looking for contributors around provider/model adapters, workflow docs, UX
polish, continuity review, sound, export, and example projects.
```

### Comment Reply

```text
It is not meant to replace every specialist tool. The goal is to be an open
workflow layer for AI video production: local project ownership, BYOK providers,
inspectable source, and a place where contributors can add adapters as models
change.
```

## Metrics to Watch

- Stars after each post.
- Clones and unique visitors in GitHub traffic.
- Issues opened by new contributors.
- Trailer click-throughs.
- Which README links people click first.
- Which provider or setup questions repeat.

Turn repeated questions into docs or starter issues.

## Weekly Maintenance Loop

1. Close or label stale issues.
2. Add one screenshot, workflow note, or short clip.
3. Convert one repeated support question into documentation.
4. Highlight one contributor-friendly task.
5. Share one progress update with a concrete before/after.

Visibility should compound through proof: demo projects, screenshots, working
adapters, clear docs, and honest release notes.
